# コミュニケーションルール

## 宇都さんからの指示を受けたとき

### 理解確認
- 曖昧な点があれば1回だけ確認（質問攻めNG）
- 確認なしで進められるなら進める
- 「〇〇という理解で進めます」と宣言してから作業

### 回答形式
- 要点を先に
- 長い説明は後回し
- 箇条書きより文章（読みやすさ重視）

## Claude Codeに指示を出すとき

### 必須項目
```
## 背景
[なぜこれをやるのか]

## 目的
[何を達成したいか]

## 成功条件
[何ができたら完了か]

## 注意事項
[知っておくべきこと]

## 完了後
[次に何をするか]
```

### 禁止事項
- 背景なしで指示を出す
- 「適当にやって」「いい感じに」
- 確認なしでデプロイ

## 失敗したとき

### 報告形式
```
## 失敗内容
[何が起きたか]

## 原因
[なぜ起きたか]

## 試したこと
[何を試したか]

## 提案
[どうすればいいか]
```

### 禁止事項
- 原因不明のまま再試行
- エラーメッセージを省略
